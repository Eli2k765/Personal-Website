# Deployment Guide

## Overview
This site is a static HTML site generated by `node build.js`. You can deploy it to any web server (Apache, Nginx, IIS) or static host (Netlify, Vercel, GitHub Pages).

## Option 1: Automated Deployment Script (Windows)

I have included a script `deploy.ps1` that uses `scp` (Secure Copy) to transfer the site to your remote server.

### Prerequisites
1.  **SSH Access**: You must have SSH access to your remote server (e.g., `user@your-server.com`).
2.  **SSH Key**: Ideally, set up SSH key authentication so you don't have to type your password every time.

### Usage
1.  Open `deploy.ps1` and edit the configuration variables:
    ```powershell
    $RemoteUser = "your_username"
    $RemoteHost = "your_server_ip_or_domain"
    $RemotePath = "/var/www/html" # Or wherever your site lives
    ```
2.  Run the script:
    ```powershell
    .\deploy.ps1
    ```

## Option 2: Manual Deployment

1.  **Build** the site locally:
    ```bash
    node build.js
    ```
2.  **Copy** the contents of the `public/` folder to your server.
    *   **Drag & Drop**: Use FileZilla or WinSCP.
    *   **Command Line**:
        ```bash
        scp -r public/* user@host:/var/www/html
        ```

## Option 3: Git Hook (Advanced)

If you want to deploy simply by pushing via Git:

1.  **On Server**: Create a bare repo.
    ```bash
    mkdir -p /var/repo/site.git
    cd /var/repo/site.git
    git init --bare
    ```
2.  **On Server**: Create a `post-receive` hook in `/var/repo/site.git/hooks/post-receive`.
    ```bash
    #!/bin/bash
    git --work-tree=/var/www/html --git-dir=/var/repo/site.git checkout -f
    ```
3.  **Local**: Add remote and push.
    ```bash
    git remote add live ssh://user@host/var/repo/site.git
    git push live main
    ```
